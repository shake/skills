### 需要覆盖的内容维度

**1. OpenClaw 是什么**
- 定位：AI Agent 编排/调度框架
- 核心定位：多部门 Agent 团队管理中枢

**2. 核心架构**
- Agent 模型：多个专业化 Agent（xhs/wechat/twitter/youtube/intel/rnd 等）
- 协作机制：shared 目录 + inbox 任务派发 + 跨部门通信
- 调度中枢：主 Agent（总经理角色）统一分发任务

**3. 核心能力**
- 任务拆解与下发
- 跨部门协作编排
- 定时任务（Heartbeat）
- 记忆系统（Memory/日志/长期记忆）
- 多频道集成（Discord 等）
- Skill 扩展机制

**4. 特色功能**
- Bridge：Agent 调用 Claude Code 的稳定中间层
- 子 Agent 并行：多任务同时执行
- 状态追踪：任务不「失踪」
- 凌晨自动运行能力

**5. 竞品对比或场景分析**
- vs 直接调 API：成本、稳定性的取舍
- vs 单 Agent 系统：多 Agent 协作的优势
- 适合场景：一人公司、多平台运营、内容矩阵

**6. 局限性与挑战**
- 陡峭的学习曲线
- Agent 协作的复杂性
- 依赖调度的可靠性

**7. 总结与建议**
- 什么样的用户适合用 OpenClaw
- 升级路径或入门建议
