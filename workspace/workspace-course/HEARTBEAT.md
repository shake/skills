# 心跳检查清单
## 检查项

### Shared 协作检查
- 读 `shared/<自己>/inbox/` — 有任务文件则按优先级处理（文件名前缀 P0/P1/P2 或 DONE）
- DONE 开头的文件是其他部门的回执，读取确认后移到 `shared/<自己>/archive/`
- 任务完成后回执写 `shared/<来源部门>/inbox/DONE-日期-简述.md`

### 进行中任务巡检
- 扫 `memory/` 最近 2 天日志（今天+昨天），找所有 `[进行中]` 标记
- 超过 24h 未更新的 `[进行中]` 任务：
  - 在本频道播报提醒
  - `sessions_spawn` 告警给 main，说明任务名和卡住时长
  - **提升到 MEMORY.md**：在 `## 未完成任务` 段落追加该条目，确保不因日志滑出窗口而丢失
- MEMORY.md 中的 `## 未完成任务` 条目完成后及时删除

### 反思巡检
- 读 `memory/` 最近 2 天日志，识别重复出现的模式
- 重复 ≥2 次的模式 → 提炼到 MEMORY.md，格式 `- [今日日期, ×N] 描述`
- 已有条目再次验证 → 只更新日期和 ×N，不新增
- MEMORY.md 超 100 行 → 精简最旧且 ×N 最低的条目
- MEMORY.md 中 ×N ≥ 3 的条目 → 标记 `[待晋升]`，周复盘时处理
- **保底写入**：若今日 `memory/YYYY-MM-DD.md` 不存在，主动创建并写入本次心跳摘要（「Heartbeat 巡检完成，无待处理任务」即可）。确保 memory/ 目录不会持续为空。
